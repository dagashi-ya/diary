Terraform実行環境構築#

前提#

  • 以下の環境で構築を実施

    • WorkSpaces（Amazon Linux release 2 (Karoo)）

事前準備#

  • AWSアカウントの作成

  • terraformからAWSにアクセスするためのアクセスキー/シークレットアクセスキー

    1. AWSコンソールにログイン

    2. サービス「IAM」を選択

    3. 左ペインからユーザーを選択し、自分のユーザー名を選択

    4. 認証情報のタブから「アクセスキーの作成」をクリック

    5. 作成されたアクセスキーとシークレットアクセスキーをコピー

構築手順#

AWS CLIインストール#

  1. AWS CLI インストールファイルDL

curl "https://awscli.amazonaws.com/awscli-exe-linux-x86_64.zip" -o ~/awscliv2.zip

  1. インストールファイル展開

unzip ~/awscliv2.zip

  1. AWS CLI インストール

sudo ~/aws/install

  1. AWS CLIへの設定情報保存

aws configure --profile {プロファイル名}

以下のように質問に答える。

AWS Access Key ID [None]: {事前準備で用意したアクセスキー}
AWS Secret Access Key [None]: {事前準備で用意したシークレットアクセスキー}
Default region name [None]: ap-northeast-1
Default output format [None]: json

Terraformインストール#

注釈

今回はtfenvというツールを使ってTerraformをインストールする

tfenvとはTerraformを複数バージョンインストールして、切り替えて使用できるツール

  1. Gitリポジトリーをclone

git clone https://github.com/tfutils/tfenv.git ~/.tfenv

  1. パスを通す

vi ~/.bashrc

以下のように追記する

export PATH="$PATH:$HOME/.tfenv/bin"

設定を読み込む

source .bashrc

  1. tfenvでTerraformのインストール可能バージョンの確認

tfenv list-remote

1.5.0-alpha20230504
1.5.0-alpha20230405
1.4.6
1.4.5
1.4.4

今回は、最新かつ安定バージョンの「1.4.6」をインストールする

  1. tfenv インストール

tfenv install 1.4.6

  1. 使用するTerraformバージョンの指定

tfenv use 1.4.6

設定を確認する

tfenv list

  1. オートコンプリートの有効化

terraform -install-autocomplete

git-secretsインストール#

注釈

git-secretsを利用する事により、AWSアクセスキー、AWSシークレットキーをgitリポジトリにコミットできないようにコミット直前にチェックできるようになる

  1. Gitリポジトリーをclone

git clone https://github.com/awslabs/git-secrets.git ~/.git-secrets

  1. git-secrets インストール

cd ~/.git-secrets/ && sudo make install

  1. git-secrets をAWS用に設定

git secrets --register-aws --global

  1. Gitのカスタム

git secrets --install ~/.git-templates/git-secrets
git config --global init.templatedir '~/.git-templates/git-secrets'
